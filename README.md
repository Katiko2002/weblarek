# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.


## Данные

две основные сущности данных: товар и покупатель.

### Тип способа оплаты

```ts
type TPayment = 'card' | 'cash';
```

`TPayment` описывает доступные способы оплаты заказа. Значение может быть только одним из двух вариантов:

`'card'` — оплата картой;
`'cash'` — оплата наличными.

### Интерфейс товара

```ts
interface IProduct {
  id: string;
  description: string;
  image: string;
  title: string;
  category: string;
  price: number | null;
}
```

`IProduct` - товар в каталоге интернет-магазина.

Поля интерфейса:

`id: string` — уникальный идентификатор товара;
`description: string` — описание товара;
`image: string` — ссылка на изображение товара;
`title: string` — название товара;
`category: string` — категория товара;
`price: number | null` — цена товара. Если цена равна `null`, товар недоступен для покупки.

### Интерфейс покупателя

```ts
interface IBuyer {
  payment: TPayment;
  email: string;
  phone: string;
  address: string;
}
```

`IBuyer` - данные покупателя, которые нужны для оформления заказа.

Поля интерфейса:

`payment: TPayment` — выбранный способ оплаты;
`email: string` — электронная почта покупателя;
`phone: string` — телефон покупателя;
`address: string` — адрес доставки.

### Тип ошибок валидации данных покупателя

```ts
type TBuyerErrors = Partial<Record<keyof IBuyer, string>>;
```

`TBuyerErrors` описывает объект с ошибками валидации данных покупателя.
Ключами объекта могут быть поля интерфейса `IBuyer`, а значениями — текст ошибки.


Если поле заполнено корректно, ошибка для него в объекте отсутствует.

## Модели данных

Модели данных отвечают только за хранение и обработку данных приложения. Они не работают с DOM-элементами, не отображают данные на странице и не выполняют запросы к серверу.

### Класс `ProductsModel`

Класс отвечает за хранение каталога товаров и товара, выбранного для подробного просмотра.

Конструктор:

```ts
constructor()
```

Конструктор не принимает параметров. При создании экземпляра класса создаётся пустой массив товаров и пустое значение выбранного товара.

Поля класса:

```ts
private items: IProduct[]
```

Хранит массив всех товаров каталога.

```ts
private selectedProduct: IProduct | null
```

Хранит товар, выбранный для подробного просмотра. Если товар не выбран, хранится `null`.

Методы класса:

```ts
setItems(items: IProduct[]): void
```

Сохраняет массив товаров в модель.

```ts
getItems(): IProduct[]
```

Возвращает массив всех товаров каталога.

```ts
getItem(id: string): IProduct | undefined
```

Возвращает товар по его идентификатору. Если товар не найден, возвращает `undefined`.

```ts
setSelectedProduct(product: IProduct): void
```

Сохраняет товар для подробного отображения.

```ts
getSelectedProduct(): IProduct | null
```

Возвращает товар, выбранный для подробного отображения. Если товар не выбран, возвращает `null`.

### Класс `BasketModel`

Класс отвечает за хранение товаров, которые пользователь добавил в корзину.

Конструктор:

```ts
constructor()
```

Конструктор не принимает параметров. При создании экземпляра класса создаётся пустой массив товаров в корзине.

Поля класса:

```ts
private items: IProduct[]
```

Хранит массив товаров, добавленных в корзину.

Методы класса:

```ts
getItems(): IProduct[]
```

Возвращает массив товаров, которые находятся в корзине.

```ts
addItem(product: IProduct): void
```

Добавляет товар в корзину.

```ts
removeItem(id: string): void
```

Удаляет товар из корзины по его идентификатору.

```ts
clear(): void
```

Очищает корзину.

```ts
getTotal(): number
```

Возвращает общую стоимость товаров в корзине. Товары с ценой `null` не увеличивают итоговую сумму.

```ts
getCount(): number
```

Возвращает количество товаров в корзине.

```ts
hasItem(id: string): boolean
```

Проверяет, есть ли товар с переданным идентификатором в корзине.

### Класс `BuyerModel`

Класс отвечает за хранение данных покупателя и их валидацию при оформлении заказа.

Конструктор:

```ts
constructor()
```

Конструктор не принимает параметров. При создании экземпляра класса данные покупателя пустые.

Поля класса:

```ts
private data: Partial<IBuyer>
```

Хранит данные покупателя. Используется `Partial<IBuyer>`, потому что пользователь может заполнять данные постепенно: сначала способ оплаты и адрес, затем email и телефон.

Методы класса:

```ts
setData(data: Partial<IBuyer>): void
```

Сохраняет данные покупателя в модель. Метод позволяет обновить только часть данных, не удаляя уже сохранённые значения.

```ts
getData(): Partial<IBuyer>
```

Возвращает сохранённые данные покупателя.

```ts
clear(): void
```

Очищает данные покупателя.

```ts
validate(): TBuyerErrors
```

Проверяет данные покупателя и возвращает объект с ошибками. Если поле заполнено корректно, ошибка для него отсутствует.

Правила валидации:

* если не выбран способ оплаты, возвращается ошибка для поля `payment`;
* если не указан адрес доставки, возвращается ошибка для поля `address`;
* если не указан email, возвращается ошибка для поля `email`;
* если не указан телефон, возвращается ошибка для поля `phone`.


## Слой коммуникации

Слой коммуникации отвечает за получение данных с сервера и отправку данных на сервер.

Класс `AppApi` - для работы с сервером.
Класс `AppApi` будет принимать в конструктор объект, соответствующий интерфейсу `IApi`, 
и вызывать его методы `get` и `post`.

Методы класса:
`getProducts(): Promise<IProductsResponse>` — выполняет GET-запрос на эндпоинт `/product/` и получает с сервера объект с массивом товаров.
`postOrder(order: TOrderRequest): Promise<IOrderResponse>` — выполняет POST-запрос на эндпоинт `/order/`, отправляет данные заказа на сервер и получает объект с подтверждением заказа.

## Слой представления View

Слой представления отвечает за отображение данных на странице и за реакцию на действия пользователя. Классы представления не хранят данные приложения и не принимают самостоятельных решений о том, как должна измениться логика приложения. При действиях пользователя классы представления генерируют события, которые обрабатываются в слое Презентера.

Все классы представления работают с готовой разметкой из `index.html`.

### Класс `Page`

Класс отвечает за главную страницу приложения.

Зона ответственности:

- отображение каталога товаров;
- отображение счётчика товаров в корзине;
- обработка нажатия на кнопку корзины;
- блокировка прокрутки страницы при открытом модальном окне.

Конструктор:

`constructor(container: HTMLElement, events: IEvents)`

Поля класса:

- `gallery: HTMLElement` — контейнер каталога товаров;
- `basketButton: HTMLButtonElement` — кнопка открытия корзины;
- `basketCounterElement: HTMLElement` — DOM-элемент счётчика товаров в корзине;
- `wrapper: HTMLElement` — основной контейнер страницы.

Методы и сеттеры класса:

- `set catalog(items: HTMLElement[])` — отображает карточки товаров в каталоге;
- `set basketCounter(value: number)` — обновляет счётчик товаров в корзине;
- `set locked(value: boolean)` — блокирует или разблокирует прокрутку страницы.

События:

- `basket:open` — генерируется при клике на кнопку корзины.

Конструктор:

`constructor(container: HTMLElement, events: IEvents)`

Методы класса:

- `render(data?: Partial<IBuyer>): HTMLElement` — возвращает разметку первой формы оформления заказа.

### Класс `ContactsForm`

Класс отвечает за второй шаг оформления заказа.

Наследуется от `Form`.

Зона ответственности:

- ввод email покупателя;
- ввод телефона покупателя;
- генерация событий изменения email и телефона;
- генерация события отправки заказа.

Конструктор:

`constructor(container: HTMLFormElement, events: IEvents)`

Поля класса:

- `emailInput: HTMLInputElement` — поле ввода email;
- `phoneInput: HTMLInputElement` — поле ввода телефона.

Методы класса:

- `setEmail(value: string): void` — отображает email;
- `setPhone(value: string): void` — отображает телефон;
- `render(data?: Partial<IBuyer>): HTMLElement` — возвращает разметку второй формы оформления заказа.

### Класс `Modal`

Класс отвечает за модальное окно.

Модальное окно не имеет дочерних классов. Вся разметка, которая отображается внутри модального окна, является самостоятельными компонентами.

Зона ответственности:

- открытие модального окна;
- закрытие модального окна;
- отображение переданного контента;
- генерация события закрытия модального окна.

Конструктор:

`constructor(container: HTMLElement, events: IEvents)`

Поля класса:

- `container: HTMLElement` — корневой элемент модального окна;
- `content: HTMLElement` — контейнер содержимого модального окна;
- `closeButton: HTMLButtonElement` — кнопка закрытия модального окна.

Методы класса:

- `setContent(content: HTMLElement): void` — устанавливает содержимое модального окна;
- `open(): void` — открывает модальное окно;
- `close(): void` — закрывает модальное окно;
- `render(): HTMLElement` — возвращает разметку модального окна.

### Класс `Success`

Класс отвечает за отображение успешного оформления заказа.

Зона ответственности:

- отображение сообщения об успешной оплате;
- отображение списанной суммы;
- генерация события закрытия окна успешной оплаты.

Конструктор:

`constructor(container: HTMLElement, events: IEvents)`

Поля класса:

- `total: HTMLElement` — элемент суммы заказа;
- `button: HTMLButtonElement` — кнопка закрытия сообщения об успешной оплате.

Методы класса:

- `setTotal(value: number): void` — отображает сумму заказа;
- `render(data: { total: number }): HTMLElement` — возвращает разметку успешного оформления заказа.

## События приложения

События используются для связи слоёв приложения. Классы представления и модели данных генерируют события, а Презентер подписывается на них и выполняет нужные действия.

### События слоя View

`basket:open` — генерируется классом `Page` при нажатии на кнопку корзины. Событие сообщает Презентеру, что пользователь хочет открыть корзину.

`card:select` — генерируется классом `CatalogCard` при клике на карточку товара в каталоге. Событие сообщает Презентеру, что пользователь выбрал товар для подробного просмотра.

`card:action` — генерируется классом `PreviewCard` при клике на кнопку действия в подробной карточке товара. Событие сообщает Презентеру, что пользователь нажал кнопку товара, а Презентер сам решает, добавить товар в корзину или удалить его.

`basket:item-remove` — генерируется классом `BasketCard` при клике на кнопку удаления товара из корзины. Событие сообщает Презентеру, какой товар нужно удалить.

`order:open` — генерируется классом `Basket` при клике на кнопку оформления заказа. Событие сообщает Презентеру, что пользователь хочет перейти к оформлению заказа.

`order:payment-change` — генерируется классом `OrderForm` при выборе способа оплаты. Событие передаёт выбранный способ оплаты.

`order:address-change` — генерируется классом `OrderForm` при вводе адреса доставки. Событие передаёт новое значение адреса.

`order:submit` — генерируется классом `OrderForm` при отправке первой формы оформления заказа. Событие сообщает Презентеру, что пользователь хочет перейти к вводу контактов.

`contacts:email-change` — генерируется классом `ContactsForm` при вводе email. Событие передаёт новое значение email.

`contacts:phone-change` — генерируется классом `ContactsForm` при вводе телефона. Событие передаёт новое значение телефона.

`contacts:submit` — генерируется классом `ContactsForm` при отправке формы контактов. Событие сообщает Презентеру, что пользователь хочет отправить заказ.

`modal:open` — генерируется классом `Modal` при открытии модального окна. Событие сообщает Презентеру, что модальное окно открыто.

`modal:close` — генерируется классом `Modal` при закрытии модального окна. Событие сообщает Презентеру, что модальное окно закрыто.

`success:close` — генерируется классом `Success` при клике на кнопку закрытия окна успешного оформления заказа. Событие сообщает Презентеру, что пользователь закрыл сообщение об успешной покупке.

### События слоя Model

`products:changed` — генерируется классом `ProductsModel` после сохранения массива товаров в модель.

`product:selected` — генерируется классом `ProductsModel` после сохранения выбранного товара для подробного просмотра.

`basket:changed` — генерируется классом `BasketModel` после изменения корзины: добавления товара, удаления товара или очистки корзины.

`buyer:changed` — генерируется классом `BuyerModel` после изменения данных покупателя или очистки данных покупателя.

## Слой Презентера

Презентер отвечает за основную логику приложения и связывает слой данных со слоем представления.

В проекте используется один Презентер, так как приложение состоит из одной страницы. Код Презентера находится в файле `src/main.ts`.

Презентер выполняет следующие задачи:

- создаёт экземпляры моделей данных;
- создаёт экземпляры классов представления;
- создаёт экземпляр класса для работы с API;
- подписывается на события моделей и представлений;
- вызывает методы моделей при действиях пользователя;
- обновляет представление при изменении данных в моделях;
- отправляет заказ на сервер.

Презентер не генерирует события самостоятельно. Он только подписывается на события, обрабатывает их и вызывает нужные методы моделей или представлений.
